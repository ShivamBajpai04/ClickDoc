QRThing Shipment API Documentation

This document provides details for integrating with the QRThing Shipment API. You can use these endpoints to create, track, update, and cancel shipments.

Base URL

https://api.qrthing.com/v1

Authentication

All requests must include a valid API key in the header:

Authorization: Bearer <api_token>

1. Create Shipment

Creates a new outbound shipment and returns the shipment ID.

Endpoint 
POST /shipments
Request 
{
    "orderId": "123456789",
    "pickupAddress": {
        "street": "221B Baker Street",
        "city": "London",
        "country": "UK"
    },
    "deliveryAddress": {
        "street": "21 Jump Street",
        "city": "LA",
        "country": "USA"
    }
}
Response 
{
  "shipmentId": "ship_673820",
  "status": "created",
  "estimatedDelivery": "2025-07-08"
}

2. Track Shipment

Return the latest status of the shipment.

Endpoint 
GET /shipments/{shipmentId}/tracking
Response 
{
  "shipmentId": "ship_673820",
  "status": "in_transit",
  "lastLocation": "New York, JFK",
  "updatedAt": "2025-07-08T15:22:00Z"
}

3. Cancel Shipment

Cancel an already created shipment (if possible).

Endpoint 
POST /shipments/{shipmentId}/cancel
Request 
{
  "reason": "customer_requested"
}
Response 
{
  "shipmentId": "ship_673820",
  "status": "cancelled"
}

4. Get Shipment Label

Generates a printable shipping label (PDF).

Endpoint 
GET /shipments/{shipmentId}/label
Response (PDF binary) 
%PDF-1.1 ... (binary) ... %%EOF

5. List Shipments

List shipments created within a given time window.

Endpoint 
GET /shipments?created_after=2025-07-01&limit=50
Response 
{
  "total": 124,
  "results": [
    {
      "shipmentId": "ship_673820",
      "status": "in_transit"
    },
    {
      "shipmentId": "ship_673821",
      "status": "created"
    }
  ]
}

Error Schema

All endpoints may return the following error model:

{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "Pickup city is required"
  }
}

HTTP Status Codes

Status Meaning
200 Success
201 Created
400 Bad Request
401 Unauthorized / Missing API Key
404 Not Found
500 Internal Server Error

Versioning

Use the api-version header to target a specific version.

api-version: 2025-05-01

If omitted, the latest stable version is used.
